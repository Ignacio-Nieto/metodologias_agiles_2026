# Ejercicio Nro: 2

## Enunciado
¿Qué es para nosotros el desarrollo de software?

- ¿Ejemplos de desarrollo de software en los que hayas participado? ¿Ejemplos que conozcas?
- ¿Cuál es el papel del desarrollador en el desarrollo de un proyecto software?
- ¿Hay otros papeles importantes en el desarrollo, aparte del desarrollador?
- ¿Cuándo se ha desarrollado un buen software?

## Resolución
Para mí, el desarrollo de software es el proceso de analizar una necesidad o un problema y transformarlo en una solución tecnológica útil. No se trata solamente de programar, sino también de pensar, diseñar, probar, corregir y mejorar un sistema para que realmente sirva a quienes lo van a usar.

He participado en proyectos de las materias de mi facultad, especialmente en Programación Orientada a Objetos, Desarrollo de Arquitectura de Software e Ingeniería de Software. En esos trabajos fui poniendo en práctica conceptos como análisis de requerimientos, diseño de clases, desarrollo de funcionalidades, organización del sistema y resolución de problemas. Además, conozco ejemplos de desarrollo de software como sistemas de gestión, aplicaciones web, apps móviles, plataformas de reservas, sistemas bancarios y programas de control de stock.

El papel del desarrollador dentro de un proyecto de software es muy importante, ya que es quien transforma los requerimientos y las ideas en una solución concreta. Su tarea no es solo escribir código, sino también comprender el problema, proponer soluciones, mantener el sistema, corregir errores y trabajar en equipo con otros participantes del proyecto.

Sí, además del desarrollador existen otros papeles importantes. Por ejemplo, el analista funcional, que ayuda a identificar las necesidades del usuario; el tester, que verifica que el sistema funcione correctamente; el diseñador UX/UI, que trabaja sobre la experiencia y la interfaz; el project manager o líder de proyecto, que organiza tiempos y tareas; y también el cliente o usuario, que aporta la visión de lo que realmente se necesita.

Considero que se ha desarrollado un buen software cuando el sistema cumple con su objetivo, resuelve el problema para el cual fue creado y funciona de manera correcta, clara y confiable. También debe ser fácil de usar, mantenerse en el tiempo y permitir mejoras o cambios cuando sea necesario.
