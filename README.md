# 📚 Metodologías de Desarrollo Ágiles 2026

## Espacio de Entrega de Ejercicios

Bienvenidos al espacio de entrega y corrección de ejercicios de la materia **Metodologías de Desarrollo Ágiles**.

---

## 🎯 Propósito de este Espacio

Este repositorio tiene como objetivo:

1. **Centralizar** todas las entregas de ejercicios de los estudiantes
2. **Facilitar** el proceso de revisión y corrección mediante herramientas automatizadas
3. **Generar** retroalimentación detallada para cada ejercicio resuelto
4. **Documentar** el progreso de aprendizaje de cada estudiante
5. **Promover** el trabajo colaborativo y la mejora continua

---

## 📁 Estructura de Entrega

### Formato de Carpeta Personal

Cada estudiante debe crear una carpeta con el siguiente formato:

```
Nombre Apellido - NúmeroDeLegajo/
```

**Ejemplo:**
```
Juan Pérez - 12345/
```

### Formato de Archivos de Ejercicios

Por cada ejercicio resuelto, crea un archivo Markdown (`.md`) con el siguiente formato:

```
Ejercicio Nro: X.md
```

**Ejemplo:**
```
Ejercicio Nro: 1.md
Ejercicio Nro: 2.md
```

### Contenido del Archivo

Cada archivo debe contener:

1. **Enunciado del ejercicio** (copiar el enunciado original)
2. **Resolución** (tu respuesta/solución al ejercicio)

**Ejemplo de estructura:**

```markdown
# Ejercicio Nro: 1

## Enunciado
Describa qué es una metodología ágil y mencione al menos 3 ejemplos.

## Resolución
[Acá va tu respuesta...]
```

## 🔄 Proceso de Entrega

### Paso 1: Fork del Repositorio

Hacé un **Fork** de este repositorio a tu cuenta de GitHub:

```
https://github.com/cursos-uai/metodologias_agiles_2026.git
```

### Paso 2: Crear tu Carpeta

En tu fork, crea tu carpeta personal siguiendo el formato indicado arriba.

### Paso 3: Subir tus Ejercicios

Agrega los archivos `.md` con tus ejercicios resueltos en tu carpeta personal.

### Paso 4: Pull Request (PR)

Cuando termines tus ejercicios, crea un **Pull Request** al repositorio original para que sean revisados.

**Título del PR:**
```
Entrega: Nombre Apellido - Legajo - Ejercicios X, Y, Z
```

---

## ✅ Proceso de Corrección

### Corrección Automatizada

Los ejercicios serán revisados por un sistema automatizado que:

1. **Verifica** la estructura y formato de la entrega
2. **Evalúa** el contenido de las respuestas
3. **Genera** un archivo de corrección detallado

### Archivo de Corrección

Por cada ejercicio entregado, se generará un archivo de corrección con el siguiente formato:

```
Ejercicio Nro: X-Corregido.md
```

**Ejemplo:**
```
Ejercicio Nro: 1-Corregido.md
```

### Contenido de la Corrección

El archivo de corrección incluirá:

- ✅ **Puntos correctos** y fortalezas de la respuesta
- ⚠️ **Áreas de mejora** con sugerencias específicas
- ❌ **Errores conceptuales** (si los hubiere) con explicaciones
- 📖 **Referencias** al material de estudio cuando corresponda
- 📝 **Nota/Calificación** (si aplica)
